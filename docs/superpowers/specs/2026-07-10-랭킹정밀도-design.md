# 유사조문 랭킹 정밀도 개선 (제목가중 + 부정어 페널티) 설계

> 작성: 2026-07-10 · 상태: 설계 승인 대기
> 대상: `simmatch.db_similar` 재정렬 점수. Phase 1·2 공용 엔진 개선(타사·표준약관 모두 적용).

## 1. 문제 (실측 진단)

문자 n-gram + BM25 블렌딩은 표면 형태 겹침에 치우쳐 두 실패 모드를 낳음:

**① 부정조문이 긍정조문을 이김** — 질의 "…보험금을 지급합니다"에 표준약관 제5조
"보험금을 지급하지 **않는** 사유"(0.575)가 제3조 "보험금의 지급사유"(0.335)보다 상위.
부지급 조문이 더 길고 `보험금`·`지급` n-gram을 더 많이 공유하며, "않는" 부정 신호를 엔진이 못 봄.

**② 변별 용어가 보일러플레이트에 묻힘** — 질의 "…납입최고를 하고 계약을 해지"에
"계약자의 임의해지"(0.353)가 정작 맞는 "납입최고(독촉)와 계약의 해지"(0.24)보다 상위.
변별 용어 "납입최고"가 후보 제목에 있으나 그 신호가 약함.

공통점: **제목이 주제 신호를 담는데 과소 반영됨.** ①은 제목의 부정어, ②는 제목의 변별 용어.

## 2. 처방 (실험으로 가중치 도출)

`db_similar` 재정렬 점수에 두 신호를 추가:

```
score = 0.7·cos(query,text) + 0.3·bm_norm          # 기존 블렌딩 (불변)
        + W_TITLE · cos(query, title)               # ① 제목 코사인 가중(조사 무관)
        − W_NEG   · [후보 제목에 부정어 && 질의엔 없음]  # ② 부정어 비대칭 페널티
        + 0.05 · [기존 exact-title 일치 보너스]        # (기존 유지)
```

- **제목 코사인 가중**: 후보 `title`을 같은 문자 n-gram TF-IDF로 벡터화해 질의와 코사인.
  조사(을/를/이/가)에 무관하게 "납입최고" 같은 변별어를 잡음. 제목의 주제 적합도를 반영.
- **부정어 페널티**: 후보 제목에 부정 마커가 있고 질의에는 없으면 감점.
  질의도 부정이면(부지급 조문 심사) 대칭적으로 페널티 없음 → 올바름.
  - 부정 마커(제목 대상, 공백 제거 후 부분일치): `지급하지않`, `지급하지아니`, `하지아니`,
    `하지않`, `않는`, `않습니다`, `못하`, `부지급`. (`제외`는 오탐 우려로 제외.)

**가중치(실험 도출, 구현 시 골든셋으로 미세조정)**: `W_TITLE = 0.35`, `W_NEG = 0.30`.
- 이 값에서: ① 표준약관 제3조 지급사유가 top-1·2로 상승(부지급 강등), ② 정답 조문 top-3 유지,
  Q1(사망)·Q3(청약철회)·STD청약철회 무회귀 — 실측 확인.

## 3. 적용 범위

- `db_similar` 한 함수의 재정렬 루프에만 추가. FTS 후보 생성·코사인·BM25 코어 불변.
- STANDARD·TERMS 양쪽에 동일 적용(doc_type 무관).
- 기존 호출부 시그니처 불변(내부 계산만 확장). 하위호환.
- 런타임 stdlib 유지(제목 코사인은 기존 `vectorize`/`cosine` 재사용, 부정어는 문자열 검사).

## 4. 검증

- **골든셋 확장**(`tests/test_golden_recall.py`):
  - ① STANDARD 사망보험금 질의 → top-1 title이 "보험금의 지급사유"(부지급 아님).
  - Q1/Q3 무회귀(기존 assert 유지), Q2는 정답 조문 top-3 포함(느슨 유지).
  - 부정어 대칭성: 부지급 질의를 넣으면 부지급 조문이 정상 상위(페널티 안 걸림).
- **단위 테스트**: 부정어 감지 함수(`_has_negation`), 제목 코사인 가중이 점수에 반영되는지(소형 픽스처).
- 회귀: 전체 스위트 green. 브라우저 스모크(표준약관 섹션 top-1이 지급사유).

## 5. 리스크

- 부정어 휴리스틱 취약성 → 마커를 제목(짧고 표준화됨)에만 적용, 대칭 설계, 골든셋으로 방어.
- 가중치 과적합 → 4개 질의로만 튜닝하지 않도록 골든셋에 부정어 대칭·표준약관 케이스 추가.
- ② Q2는 완전 해결 아님(임의해지 표면겹침) — top-3 노출로 수용, 추가 튜닝은 별도.

## 6. 후속

- 용어 사전 기반 부스팅(납입최고·청약철회 등 표준 조문명 사전) — 필요 시 별도.
- 이 개선은 Phase 3(감독규정 연결)이 같은 엔진을 쓰므로 그 정밀도에도 기여.
