# 감독규정·법령 연결 (Phase 3) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 보험업감독규정·보험업법(시행령·시행규칙)을 `doc_type='REG'`로 적재하고, 유사조문 결과에 "관련 감독규정·법령" 섹션을 추가해 초안 조문에 규범 근거를 자동 연결한다.

**Architecture:** Phase 1·2 엔진·검색앱 재사용. 규정을 REG 코퍼스로 적재, `db_similar(doc_type='REG')`로 매핑(제목가중이 규정 제목=주제라벨을 지렛대로 활용). UI는 2섹션→3섹션 일반화.

**Tech Stack:** Python 3.14, SQLite(FTS5), pytest. 런타임 엔진·검색앱은 표준 라이브러리만.

## Global Constraints

- 규정 조문번호는 **`제N-M조`**(감독규정) + `제N조[의K]`(법령) 혼재 → `clause_split`이 둘 다 파싱해야 함. 이 변경은 공용 함수라 **기존 약관·표준약관 파싱 무회귀**가 필수.
- `db_similar` 코어·시그니처 불변(Phase 2에서 추가한 `doc_type` 필터 재사용). 런타임 stdlib.
- REG 발행처 member_cd: 감독규정 `REG_GD`, 보험업법 `REG_LAW`, 시행령 `REG_LAWE`, 시행규칙 `REG_LAWR`. 발행처명이 UI 라벨.
- `version_label`에 시행일 명시(규범 버전).
- 매핑 품질은 **적재 후 실측 게이트**로 판정 — 약하면 주제사전은 후속(본 계획 범위는 유사도 매핑까지).
- 테스트는 bare import, `tests/conftest.py`가 `src/` 경로 추가(기존).
- 경로 `~/yakgwan-db`, 스크립트는 `src/`에서 `.venv/bin/python`. 코드 스타일은 기존 `src/*.py`.

```python
#!/usr/bin/env python3
"""감독규정·법령 평문 → DB 적재(doc_type=REG). ingest_standards 로직 재사용.

사용법: .venv/bin/python src/ingest_regulations.py
"""
import pathlib

from common import ROOT
from ingest_standards import ingest
from standards_prep import load_manifest, to_plaintext
from common import open_db

MANIFEST = ROOT / "catalog" / "regulations.json"


def main():
    conn = open_db()
    for e in load_manifest(str(MANIFEST)):
        txt = to_plaintext(e)
        n = ingest(conn, e, txt)
        print(f"  [{e['key']}] {e['name']} ({e['version_label']}) — 조문 {n}개 적재")
    total = conn.execute("SELECT COUNT(*) FROM documents WHERE doc_type='REG'").fetchone()[0]
    print(f"완료: REG 문서 {total}건")
    conn.close()


if __name__ == "__main__":
    main()
```
